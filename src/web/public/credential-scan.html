Credential Scan

Scan a known list of hosts, authenticating each with its own username and password from a CSV file. Read-only — each row's credentials are used only for that row's connection attempt and are never stored or logged.

Targets

username and password may be blank — that row is scanned without attempting to authenticate. A header row is fine and skipped automatically. If a password itself contains a comma, wrap that field in double quotes. The file is read entirely in the browser and parsed here; it's never uploaded as a raw file, only the parsed host/port/username/password values are sent when you start the scan.
